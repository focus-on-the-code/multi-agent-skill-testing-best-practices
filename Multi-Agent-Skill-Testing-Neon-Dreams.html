Mission origin

Why This Test Lab Exists

It began with one practical question: could the Agentic Project Kickoff skill guide a fresh agent correctly without being shown what “correct” was supposed to look like?

Agentic Project Kickoff

A reusable facilitation workflow for starting software, apps, automations, AI workflows, and agentic-coding projects with deliberate discovery before implementation.

  • Adapts its questions to the project.
  • Creates project-specific planning and workspace documents.
  • Preserves decisions, assumptions, and open questions.
  • Builds shared understanding without rushing into code.
1

Remove the answer key

A fresh child agent received the skill, a natural request, and a disposable workspace—not our expected files, conclusions, or recommended fixes.

2

Simulate a real owner

Agent 1 conducted a kickoff for a children’s basic-physics website, supplying simple owner answers under an at-most-five-question test control.

3

Challenge the evidence

Agent 1 captured the run. Agent 2 independently reviewed the skill and artifacts, separating behavior the evidence proved from timing it could not prove.

4

Convert findings into upgrades

The skill gained narrow approval semantics and “Current Discovery Summary.” The test method gained live parallel review, gate snapshots, timestamps, and hashes.

WHY IT MATTERS: the result is both a clearer skill for humans and a stronger way to evaluate skills. We kept test evidence behind the scenes and rejected repetitive human validation paperwork.

Core power system

Four Rules. Zero Guesswork.

These constraints make the result meaningful instead of merely impressive-looking.

♢

Split the roles

The agent using the skill cannot also be its own independent project owner and evaluator.

◌

Shield context

The child gets the skill and natural request—not files, fixes, expected outputs, or conclusions.

▦

Keep raw proof

Reports interpret. Transcripts, artifacts, trees, timestamps, and hashes demonstrate.

⇄

Separate controls

A five-question cap bounds the test. It does not become a rule of the skill.

The timing upgrade

Agent 2 Watches It Happen

A final folder cannot prove when files appeared. The reviewer must capture the run while the gates are still closed.

Neutral event channel

  1. Agent 1 relays the exact child response.
  2. Agent 2 inspects before approval.
  3. Agent 2 snapshots tree, hashes, and time.
  4. Agent 1 sends narrow approval.
  5. Agent 2 records first visible expansion.

The child stays clean: events flow to the reviewer, never the answer key to the child.

Case-study showdown

Two Reports. Better Truth.

Agent 1 proved the workflow could work. Agent 2 showed where the retained evidence and generated decisions still needed sharper boundaries.

Pass + caveats

Agent 1 · Behavioral

  • Fresh child honored the staged flow.
  • Two useful discovery questions were enough.
  • Only the initial set existed before approval.
  • Expansion stayed planning-only.
  • No prohibited project docs appeared.
VS
Conditional pass

Agent 2 · Independent

  • Final artifacts were structurally strong.
  • Final state alone could not prove timing.
  • The question cap was not a skill rule.
  • Expansion approval blurred decision status.
  • Hashes could prove source integrity without Git.
SKILL UPGRADES: expansion approval does not approve proposed decisions · use the heading “Current Discovery Summary.”
TEST UPGRADE: start the live parallel reviewer before the child and retain immutable gate evidence.
Threat catalog

Defeat the Test Villains

Most false confidence comes from role confusion, context leakage, or evidence captured too late.

🪞

Self-review loop

Conductor becomes the system under test.

🧠

Context leak

Expected outputs reach the child prompt.

⏱

Late reviewer

Agent 2 arrives after timing evidence is gone.

📁

Final-tree fallacy

Final artifacts are treated as proof of staging.

🎭

Narrative proof

A polished report substitutes for raw evidence.

5️⃣

Invented rule

A test cap is scored as a skill requirement.

✅

Approval creep

Continue becomes approve every proposal.

Silent stall

Human paperwork
